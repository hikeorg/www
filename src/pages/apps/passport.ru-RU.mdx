# <Passport size={96} /> Паспорт

Паспорт разработан, чтобы помочь вашей компании постепенно внедрять политику лицензирования, проверяя все ваши лицензии на зависимости, мы гарантируем, что ваши проекты не попадут в какие-либо юридические ловушки.

## Параметры конфигурации

### Check

```tsx
check: boolean = false;
```

Если установлено значение «true», будет создана проверка с использованием GitHub Checks API. Вы можете использовать проверку, чтобы предотвратить слияние, если какая-то лицензия не соответствует требованиям.

### Skip

```tsx
skip: boolean = true;
```

Если установлено значение «true», это позволит пропустить неудачную проверку, когда среди изменений запроса на вытягивание будет обнаружена оскорбительная лицензия.

### Allow

```tsx
allow: string[] = ["MIT"]
```

Массив действительных [лицензий или выражений SPDX](https://spdx.org/licenses/), которые вы хотите разрешить в своем проекте.

### Ignore

```tsx
ignore: string[]
```

Массив зависимостей, которые вы хотите игнорировать, если они не соответствуют вашим разрешающим правилам. Они должны быть предоставлены точно так же, как они появляются в любом из ваших файлов `package.json`.

## Примеры использования

### Разрешить только MIT

```yaml
check: true
allow:
  - MIT
```

Эта конфигурация создаст неудачную проверку запроса на вытягивание, которую вы можете использовать, чтобы запретить соавторам объединять свои изменения, если какая-либо из лицензий ваших зависимостей не является ни MIT, ни ISC.

### Игнорирование зависимостей

Иногда у вас нет времени искать альтернативу или вы просто не можете избежать использования зависимости с ограниченной лицензией. В этих ситуациях вы можете игнорировать вызывающие нарушение зависимости, добавив их в список «игнорировать»:

```yaml
check: true
allow:
  - MIT
ignore:
  - evilcorp@^11.1.0
```

Обязательно добавьте зависимость точно так, как определено в вашем файле `package.json`.
